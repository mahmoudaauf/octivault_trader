CRITICAL ISSUE RESOLVED
════════════════════════════════════════════════════════════════════════════════

Problem:      System double-counted position value in NAV calculation
Root Cause:   Hydration called record_trade() which modified capital ledger
Impact:       NAV incorrect, startup fails, portfolio can't trade
Severity:     🔴 CRITICAL

FIXES APPLIED
════════════════════════════════════════════════════════════════════════════════

─────────────────────────────────────────────────────────────────
  File:     core/exchange_truth_auditor.py
  Method:   _hydrate_missing_positions() (Line 1082)
  Change:   Removed _apply_recovered_fill() call
            Added direct: ss.positions[sym] = {...}
  Benefit:  ✅ Capital ledger unchanged
            ✅ Position value not double-counted
            ✅ Entry price deferred (portfolio_manager handles PnL)

Fix #2: Shadow Mode NAV Uses Wallet Value Only
─────────────────────────────────────────────────
  File:     core/shared_state.py
  Method:   get_nav_quote() (Line 1057)
  Change:   Added shadow mode detection
            Early return: if is_shadow_mode: return nav
  Benefit:  ✅ NAV = wallet_value in shadow mode
            ✅ No position addition (prevents double-count)
            ✅ Normal mode unchanged
