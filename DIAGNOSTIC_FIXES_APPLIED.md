## Diagnostic Flowchart

```
If seeing:
    [TrendHunter] Buffered BUY
    BUT NOT
    [AgentManager:NORMALIZE]
    
    → Issue: generate_signals() not being called
      Action: Check if TrendHunter is registered in AgentManager
              Check if collect_and_forward_signals() is being invoked

───────────────────────────────────────

If seeing:
    [AgentManager:NORMALIZE] Normalizing X raw signals
    BUT NOT
    [AgentManager:NORMALIZE] ✓ Successfully normalized X intents
    
    → Issue: Signals failing validation in _normalize_to_intents()
      Possible causes:
        - Symbol not in USDT quote pairs
        - Action not "BUY" or "SELL"
        - Confidence < 0.50 minimum
      Action: Check first signal in normalization log:
              "[_normalize_to_intents] provided X raw signals but NONE passed"

───────────────────────────────────────

If seeing:
    [AgentManager:NORMALIZE] ✓ Successfully normalized X intents
    BUT NOT
    [AgentManager:SUBMIT] Publishing X intents
    
    → Issue: Batch not being extended or check failing
      Action: Check if `intents` is being properly passed to submit_trade_intents()

───────────────────────────────────────

If seeing:
    [AgentManager:SUBMIT] Publishing X intents
    BUT NOT
    [AgentManager] Published X trade intent events
    
    → Issue: Event bus publishing failing
      Action: Check if event_bus is properly initialized
              Check event_bus.publish() error logs

───────────────────────────────────────

If seeing:
    [MetaController:RECV_SIGNAL] Received signal for BTCUSDT from TrendHunter
    BUT NOT
    [MetaController:RECV_SIGNAL] ✓ Signal cached
    
    → Issue: SignalManager.receive_signal() rejecting signals
      Possible causes:
        - Symbol validation failed
        - Confidence below minimum threshold
        - Quote token validation failed
      Action: Check SignalManager debug logs for rejection reason

───────────────────────────────────────

If seeing:
    [Meta:DRAIN:ENTRY] Entering _drain_trade_intent_events
    BUT NOT
    [Meta:DRAIN] ⚠️ DRAINED X events
    
    → Issue: Queue empty or subscription failed
      Action: Check if events are actually being published to bus
              Check if subscription was successful

───────────────────────────────────────

If seeing:
    None of the above diagnostic logs
    
    → Issue: One of these:
        1. collect_and_forward_signals() not being called
        2. TrendHunter not registered in AgentManager
        3. Strategy agent loop not running
        4. evaluate_and_act() not being invoked
```
