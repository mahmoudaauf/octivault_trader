```
Signal from cache
    ↓
[Meta:SIGNAL_INTAKE] - "Retrieved N signals"
    ↓
For each signal:
  [Meta:GATE_TRACE] - "Processing SYM ACTION (conf=X.XX)"
    ↓
  Apply filters:
    ├─ [Meta:GATE_DROP_RECOVERY] - "Dropped at RECOVERY gate"
    ├─ [Meta:GATE_DROP_TRADEABILITY] - "Dropped at CONFIDENCE gate"
    ├─ [Meta:GATE_DROP_ONE_POSITION] - "Dropped at POSITION gate"
    └─ Many other gates...
    ↓
  If passed all: [Meta:GATE_PASSED] - "PASSED ALL GATES"
    ↓
[Meta:AFTER_FILTER] - "valid_signals has N symbols"
    ↓
If N = 0: [Meta:DEADLOCK_DIAGNOSTIC] - "Explains why nothing passed"
```

## Quick Lookup

### Log Message Reference

| Message | Meaning | If You See It |
|---------|---------|---------------|
| `[Meta:SIGNAL_INTAKE]` | Signals from cache | Check count - should be > 0 |
| `[Meta:GATE_TRACE]` | Signal enters filter | Count should match SIGNAL_INTAKE |
| `[Meta:GATE_DROP_RECOVERY]` | Capital recovery blocks BUY | Exit recovery mode |
| `[Meta:GATE_DROP_TRADEABILITY]` | Confidence too low | Raise TrendHunter conf or lower floor |
| `[Meta:GATE_DROP_ONE_POSITION]` | Position already open | Wait for exit or check position state |
| `[Meta:GATE_PASSED]` | Signal approved! | Great! It will become decision |
| `[Meta:AFTER_FILTER]` | Final signal count | 0 = all dropped, >0 = some made it |
| `[Meta:DEADLOCK_DIAGNOSTIC]` | Explains deadlock | Read explanation for root cause |

## Common Scenarios

### Scenario 1: "Confidence Too Low"
**Log:** `[Meta:GATE_DROP_TRADEABILITY] ... conf=0.45 floor=0.55 ...`

**Root Cause:** TrendHunter signals have confidence 0.45 but gate requires 0.55

**Solutions:**
1. Check mode - might be in RECOVERY with elevated floor
2. Check TrendHunter confidence calculation
3. Check market conditions - signals might be poor quality
4. Lower confidence floor in settings

### Scenario 2: "Capital Recovery Active"
**Log:** `[Meta:GATE_DROP_RECOVERY] ... CAPITAL_RECOVERY gate ...`

**Root Cause:** Bot is in capital recovery mode, non-bootstrap BUYs suppressed

**Solutions:**
1. Exit capital recovery mode (check shared_state settings)
2. Wait for recovery to complete naturally
3. Generate bootstrap signals to escape FLAT state

### Scenario 3: "No Signals Generated"
**Log:** `[Meta:SIGNAL_INTAKE] Retrieved 0 signals from cache`

**Root Cause:** TrendHunter isn't generating signals

**Solutions:**
1. Check TrendHunter logs
2. Verify market data is flowing (OHLCV cache)
3. Restart TrendHunter agent
4. Check agent configuration

### Scenario 4: "Signals Passing"
**Log:** `[Meta:GATE_PASSED] OPNUSDT BUY PASSED ALL GATES ...`

**Status:** ✅ Signals ARE flowing!

**Next Step:** Check ExecutionManager logs to see if decisions are being executed
