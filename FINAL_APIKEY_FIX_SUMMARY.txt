                              THE PROBLEM
════════════════════════════════════════════════════════════════════════════════

Binance WS API v3 was rejecting our authentication with:

  Error Code 1008: POLICY_VIOLATION
  
  Root Cause: apiKey was NOT in the params dictionary
  
  What was being sent:
  {
    "params": {
      "timestamp": 1772321380047,
      "signature": "..."
    }
  }
  
  What should have been sent:
  {
    "params": {
      "apiKey": "YOUR_API_KEY",           ← MISSING
      "timestamp": 1772321380047,
      "signature": "..."
    }
  }

Result: Server rejected EVERY attempt with 1008 error → cascading fallback

════════════════════════════════════════════════════════════════════════════════
                              THE SOLUTION
════════════════════════════════════════════════════════════════════════════════

Add apiKey to params BEFORE calculating the signature.

This ensures:
  1. apiKey is in the params sent to the server
  2. apiKey is included in the signature calculation
  3. Query string becomes: "apiKey=...&timestamp=..."
  4. Server validates and accepts (200 OK)
