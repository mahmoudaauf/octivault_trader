## ROOT CAUSE IDENTIFIED

In `core/meta_controller.py` line 10948, the decision-building logic was checking the wrong budget metric:

```python
# The Broken Logic:
agent_budget = _wallet_budget_for(agent_name)  # Check: How much budget does agent still have?
# Answer at decision time: $0 (was all allocated in prior phase)

if agent_budget >= significant_position_usdt:   # Check: 0 >= 25? FALSE
    add_to_decisions()  # This code path never executes
```

**What Went Wrong:**
The system allocates capital in Phase 1, but then in Phase 2 it checks if the agent still has remaining budget. By that time, ALL budget was intentionally allocated away, so the check fails for every signal.

It's like saying: "I gave you $30 for this trade" → "OK" → "Wait, do you still have money?" → "No" → "Trade rejected!"

---

## THE FIX

Changed the logic to check the **signal's allocated amount** instead of the **agent's remaining balance**:

```python
# The Fixed Logic:
signal_planned_quote = float(best_sig.get("_planned_quote") or 0.0)
# Gets: $30 (what was explicitly allocated to this signal)

if signal_planned_quote <= 0:
    signal_planned_quote = _wallet_budget_for(agent_name)  # Fallback only if not allocated

if signal_planned_quote >= significant_position_usdt:   # Check: 30 >= 25? TRUE
    add_to_decisions()  # This code path executes ✅
```

**Why It Works:**
Each signal carries `_planned_quote` from the allocation phase. This is authoritative and doesn't change. We check it directly instead of asking the agent "do you still have money?"

## WHY THIS HAPPENED

The codebase has two distinct budget concepts:
1. **Agent's Remaining Budget**: How much the agent can still distribute
2. **Signal's Allocated Budget**: How much the agent DID distribute to this signal

The bug confused these two concepts. In Phase 2 (decision building), the system needed to check #2 (allocated) but was checking #1 (remaining), which by that time was always exhausted.

The fix correctly identifies that we should check what was allocated, not what's remaining.
