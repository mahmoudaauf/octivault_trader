# Signal Processing Diagnostic Guide

## Overview
The MetaController's `_build_decisions()` method filters incoming signals through multiple gates before converting them to trading decisions. When signals are generated by TrendHunter but no decisions are made, it means ALL signals were dropped by filters.

## Added Diagnostic Logging

### 1. Signal Intake
**Log Message:** `[Meta:SIGNAL_INTAKE]`
**Location:** Line 9475 in meta_controller.py
**What it shows:** All signals retrieved from SignalManager cache
**Example:**
```
[Meta:SIGNAL_INTAKE] Retrieved 5 signals from cache: [('OPNUSDT', 'BUY', 0.77), ('BARDUSDT', 'BUY', 0.65), ...]
```
**Interpretation:** If this is empty or low, signals aren't being generated/cached properly

### 2. Gate Tracing
**Log Message:** `[Meta:GATE_TRACE]`
**Location:** Line 9598 in meta_controller.py
**What it shows:** Each signal as it enters the filtering pipeline
**Example:**
```
[Meta:GATE_TRACE] Processing OPNUSDT BUY from TrendHunter (conf=0.77)
```
**Interpretation:** Shows which signals enter the filter chain and their properties

### 3. Gate Drops (Specific)
Four gate drop messages show WHERE signals are being rejected:

#### 3a. Capital Recovery Gate Drop
**Log Message:** `[Meta:GATE_DROP_RECOVERY]`
**Location:** Line 9611
**Meaning:** BUY signal rejected because bot is in capital recovery mode (and signal is not bootstrap)
**Example:**
```
[Meta:GATE_DROP_RECOVERY] OPNUSDT BUY dropped at CAPITAL_RECOVERY gate (not bootstrap)
```

#### 3b. Tradeability Gate Drop (Confidence Floor)
**Log Message:** `[Meta:GATE_DROP_TRADEABILITY]`
**Location:** Line 9679
**Meaning:** BUY signal rejected because confidence is below required floor
**Example:**
```
[Meta:GATE_DROP_TRADEABILITY] OPNUSDT BUY dropped at TRADEABILITY gate: conf=0.45 floor=0.55 gate=MODE_RECOVERY
```
**Analysis:**
- Check if `conf` < `floor`
- Check the `gate` reason (could be MODE_RECOVERY, AGGRESSIVE_FLOOR, etc.)

#### 3c. One-Position Gate Drop
**Log Message:** `[Meta:GATE_DROP_ONE_POSITION]`
**Location:** Line 9871
**Meaning:** BUY signal rejected because symbol already has an open position
**Example:**
```
[Meta:GATE_DROP_ONE_POSITION] OPNUSDT BUY dropped at ONE_POSITION_PER_SYMBOL gate (qty=0.100000)
```
**Analysis:**
- Shows existing position quantity
- Only relevant if you SEE this message AND portfolio is supposedly flat

### 4. Gate Passed
**Log Message:** `[Meta:GATE_PASSED]`
**Location:** Line 9993
**What it shows:** Signal successfully passed ALL filtering gates and was added to valid_signals
**Example:**
```
[Meta:GATE_PASSED] OPNUSDT BUY PASSED ALL GATES and ADDED to valid_signals (conf=0.77 agent=TrendHunter)
```
**Interpretation:** If you see this, the signal will become a decision (assuming later filtering doesn't reject it)

### 5. After-Filter Summary
**Log Message:** `[Meta:AFTER_FILTER]`
**Location:** Line 10002
**What it shows:** Complete state of valid_signals_by_symbol after all filtering
**Example:**
```
[Meta:AFTER_FILTER] valid_signals_by_symbol has 2 symbols with signals: 
  {'OPNUSDT': [('BUY', 0.77)], 'BARDUSDT': [('BUY', 0.65)]}
```
**Interpretation:**
- If this is 0 symbols, all signals were dropped
- If this is >0, signals made it through and will become decisions

### 6. Deadlock Diagnostic
**Log Message:** `[Meta:DEADLOCK_DIAGNOSTIC]`
**Location:** Line 10005
**What it shows:** Error message explaining why no signals passed filters
**Example:**
```
[Meta:DEADLOCK_DIAGNOSTIC] 🔴 NO SIGNALS PASSED FILTERS! 
  all_signals=5, signals_by_sym=5 (pre-filter), valid_signals_by_symbol=0 (post-filter). 
  LIKELY CAUSES: TRADEABILITY gate dropped by conf floor, CAPITAL_RECOVERY suppressed BUYs, 
  ONE_POSITION_GATE blocked existing symbols, or PROBING gate blocked new symbols. 
  Check logs for [Meta:GATE_DROP_*] messages to identify which gate(s) are filtering.
```
**Interpretation:**
- If you see this, look for [Meta:GATE_DROP_*] messages to find the culprit
- Shows counts of signals at each stage

## Diagnostic Workflow

### Step 1: Check Signal Generation
```bash
grep "\[Meta:SIGNAL_INTAKE\]" logs/clean_run.log | tail -10
```
- If you see signals, TrendHunter is working
- If empty or not found, strategy isn't generating signals

### Step 2: Trace Gate Flow
```bash
grep "\[Meta:GATE_TRACE\]" logs/clean_run.log | head -20
```
- Shows signals entering the filter
- Compare count with SIGNAL_INTAKE count

### Step 3: Find What's Dropping Signals
```bash
grep "\[Meta:GATE_DROP_\|Meta:GATE_PASSED" logs/clean_run.log | grep -E "OPNUSDT|BARDUSDT"
```
- Look for pattern of drops vs passes for each symbol
- Example output:
  ```
  [Meta:GATE_TRACE] Processing OPNUSDT BUY from TrendHunter (conf=0.77)
  [Meta:GATE_DROP_TRADEABILITY] OPNUSDT BUY dropped at TRADEABILITY gate: conf=0.77 floor=0.85 gate=AGGRESSIVE_FLOOR
  ```
  → Signals ARE being generated, but ALL are too low confidence for current mode

### Step 4: Check After-Filter State
```bash
grep "\[Meta:AFTER_FILTER\]" logs/clean_run.log | tail -5
```
- Shows final count of signals that made it through
- If this is always 0, check the DEADLOCK_DIAGNOSTIC message

### Step 5: Analyze the Deadlock Message
```bash
grep "\[Meta:DEADLOCK_DIAGNOSTIC\]" logs/clean_run.log | tail -5
```
- Explains which gate(s) are most likely dropping signals
- Use this to guide which gate to investigate

## Common Scenarios

### Scenario 1: Signals Generated But All Dropped by Confidence Floor
**Log Pattern:**
```
[Meta:SIGNAL_INTAKE] Retrieved 5 signals from cache: ...
[Meta:GATE_TRACE] Processing OPNUSDT BUY from TrendHunter (conf=0.50)
[Meta:GATE_DROP_TRADEABILITY] OPNUSDT BUY dropped at TRADEABILITY gate: conf=0.50 floor=0.55 gate=MODE_RECOVERY
[Meta:GATE_DROP_TRADEABILITY] BARDUSDT BUY dropped at TRADEABILITY gate: conf=0.52 floor=0.55 gate=MODE_RECOVERY
...
[Meta:AFTER_FILTER] valid_signals_by_symbol has 0 symbols with signals: {}
[Meta:DEADLOCK_DIAGNOSTIC] ... TRADEABILITY gate dropped by conf floor ...
```
**Root Cause:** Bot is in recovery mode with elevated confidence floor (0.55), but strategy signals are only 0.50-0.52
**Solution:** Either wait for signals with higher confidence, or adjust recovery mode settings, or check TrendHunter's confidence calculation

### Scenario 2: Signals Generated But Dropped by Capital Recovery
**Log Pattern:**
```
[Meta:SIGNAL_INTAKE] Retrieved 5 signals from cache: ...
[Meta:GATE_TRACE] Processing OPNUSDT BUY from TrendHunter (conf=0.77)
[Meta:GATE_DROP_RECOVERY] OPNUSDT BUY dropped at CAPITAL_RECOVERY gate (not bootstrap)
[Meta:GATE_DROP_RECOVERY] BARDUSDT BUY dropped at CAPITAL_RECOVERY gate (not bootstrap)
...
[Meta:DEADLOCK_DIAGNOSTIC] ... CAPITAL_RECOVERY suppressed BUYs ...
```
**Root Cause:** Capital recovery mode is active and ALL BUYs are suppressed (unless they're bootstrap signals)
**Solution:** Either exit capital recovery mode, or wait for bootstrap signal generation

### Scenario 3: No Signals Generated At All
**Log Pattern:**
```
[Meta:SIGNAL_INTAKE] Retrieved 0 signals from cache: []
[Meta:AFTER_FILTER] valid_signals_by_symbol has 0 symbols with signals: {}
[Meta:DEADLOCK_DIAGNOSTIC] all_signals=0, signals_by_sym=0 ...
```
**Root Cause:** TrendHunter isn't generating signals
**Solution:** Check TrendHunter logs, verify market data is being received, check agent configuration

### Scenario 4: Signals Made It Through!
**Log Pattern:**
```
[Meta:SIGNAL_INTAKE] Retrieved 3 signals from cache: ...
[Meta:GATE_TRACE] Processing OPNUSDT BUY from TrendHunter (conf=0.77)
[Meta:GATE_PASSED] OPNUSDT BUY PASSED ALL GATES and ADDED to valid_signals (conf=0.77 agent=TrendHunter)
[Meta:AFTER_FILTER] valid_signals_by_symbol has 1 symbols with signals: {'OPNUSDT': [('BUY', 0.77)]}
```
**Root Cause:** No issue! Signals are passing through filters
**Next Step:** Check if decisions are being made from these signals (look for execution logs)

## Summary

If you see:
- ✅ `[Meta:GATE_PASSED]` messages → Signals ARE making it through, check execution
- ❌ `[Meta:GATE_DROP_TRADEABILITY]` for all signals → Confidence floor is the issue
- ❌ `[Meta:GATE_DROP_RECOVERY]` for all signals → Capital recovery is suppressing BUYs
- ❌ `[Meta:SIGNAL_INTAKE]` with 0 signals → No signals generated by strategy
- ❌ `[Meta:DEADLOCK_DIAGNOSTIC]` message → All signals were dropped, check which gate
