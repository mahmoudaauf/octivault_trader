## Phase 5 At a Glance

### The Problem
Concentration risk enforced AFTER execution → Oversized positions → Rebalancing conflicts → Deadlock

### The Solution
Concentration risk enforced BEFORE execution → Safe sized positions → No rebalancing needed → No deadlock

### The Implementation
```python
# In get_position_sizing():
if nav > 0:
    max_position = nav * sizing["max_position_pct"]
    current_value = current_position_value or 0.0
    headroom = max_position - current_value
    adjusted_quote = min(original_quote, headroom)
```

### The Result
Positions never exceed concentration limits → System stable → Deadlock impossible

## Common Questions & Answers

### Q: Where is Phase 5 code implemented?
**A**: core/capital_governor.py, lines 274-370 in get_position_sizing() method

### Q: Do I need to update all call sites?
**A**: Recommended but not required (backward compatible). Recommended for full functionality.

### Q: What's the new method signature?
**A**: `get_position_sizing(nav, symbol="", current_position_value=0.0)`

### Q: What parameters do I pass?
**A**: nav (account NAV), symbol (trading pair), current_position_value (in USD)

### Q: How do I get current_position_value?
**A**: See **⚡_PHASE_5_INTEGRATION_GUIDE.md** → "Pattern: How to Fetch current_position_value"

### Q: What does the log [CapitalGovernor:ConcentrationGate] mean?
**A**: Quote was capped due to concentration limit. See monitoring section in any guide.

### Q: Is this backward compatible?
**A**: Yes! current_position_value defaults to 0.0 if not provided.
