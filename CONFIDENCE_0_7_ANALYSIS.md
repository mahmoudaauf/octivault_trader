# Confidence 0.7 in the System - Analysis

## The Question
**Why is confidence always 0.7?**

## The Answer
**Confidence is NOT hardcoded to 0.7 in `tp_sl_engine.py`**

The value 0.7 appears in several places, but it represents **minimum thresholds**, not hardcoded assignments.

---

## Where 0.7 Appears in the System

### 1. **meta_controller.py** (Line 1354)
```python
self._tier_a_conf = float(self._cfg("TIER_A_CONFIDENCE_THRESHOLD", 0.70))
```
- This is a **minimum threshold** for Tier A trades
- Signals with confidence < 0.70 are rejected
- Signals with confidence >= 0.70 are accepted
- **Not a hardcoded value being assigned**

### 2. **execution_manager.py** (Line 2309)
```python
confidence_adjustment = 0.7  # 30% discount for high confidence
```
- This is a **multiplier** for capital calculation
- It represents: 0.7 = 30% discount (scaling factor)
- Not an absolute confidence value

### 3. **config.py** (Lines 1220, 1224)
```python
self.CONFIDENCE_THRESHOLD = float(os.getenv("CONFIDENCE_THRESHOLD", "0.75"))
self.TIER_A_CONFIDENCE_THRESHOLD = float(os.getenv("TIER_A_CONFIDENCE_THRESHOLD", "0.75"))
```
- Configuration defaults (can be overridden by environment variables)
- Used as gates/filters, not hardcoded values

### 4. **Documentation Files**
Various markdown files reference 0.70 as a minimum signal quality standard, not an assigned value

---

## How Confidence Actually Works

### Origin
Confidence comes from **signal generators** (agents):

```python
# From agents (e.g., TrendHunter, DipSniper, etc.)
action, confidence, reason = await self._generate_signal(symbol, is_ml_capable=is_ml_capable)
await self._submit_signal(symbol, act, float(confidence), reason)
```

Confidence values are generated by each agent based on its own analysis:
- **0.55** = Low confidence
- **0.70** = Medium confidence  
- **0.85** = High confidence
- **0.95** = Very high confidence

### Filtering
The system then **filters** these signals:

```python
# meta_controller.py: Check confidence against threshold
if action_upper == "BUY":
    min_conf = float(self.min_conf)
    if float(confidence) < min_conf:
        # REJECT signal
        return
    # ACCEPT signal
```

### What Gets Stored
If a signal passes filtering, its actual confidence value is stored:

```python
# From meta_controller.py line 3481
confidence = float(directive.get("confidence", 0.75))  # Default fallback
```

## Why You Might See "0.7" Consistently

Possible reasons if all signals appear to have 0.7 confidence:

1. **Signal generators are all outputting ~0.70**
   - Check agent code (trend_hunter.py, dip_sniper.py, etc.)
   - Each agent has its own confidence calculation
   
2. **Filtering is rounding/clipping values**
   - If confidence < 0.70 is being rejected
   - And you're only seeing accepted signals
   - They appear clustered at/above 0.70

3. **Configuration default**
   - If agents aren't calculating confidence
   - They might default to 0.75 or 0.70 from config
   - Check `getattr(config, "DEFAULT_SIGNAL_CONFIDENCE", 0.75)`

4. **ML model output**
   - If using model predictions
   - The model might be outputting ~0.70 for most predictions
   - Check model calibration and training

## How to Debug This

### Step 1: Check Agent Code
Look at how confidence is calculated in agents:

```bash
# Find all confidence calculations
grep -r "confidence = " agents/
grep -r "confidence =" core/meta_controller.py
```

### Step 2: Check Signal Logs
Add detailed logging to see what confidence values are generated:

```python
# In meta_controller._execute_decision or agent code:
self.logger.info(
    "Signal confidence=%s (from agent) → threshold=%.2f → %s",
    confidence,
    min_conf,
    "PASSED" if confidence >= min_conf else "REJECTED"
)
```

### Step 3: Check for Hardcoded Defaults
Search for assignments:

```bash
grep -n "confidence.*=.*0.7" core/*.py
grep -n "confidence.*=.*0.75" core/*.py
grep -n "DEFAULT.*CONFIDENCE" core/*.py
```

### Step 4: Monitor Signal Flow
Add instrumentation:

```python
# In each agent's _submit_signal():
```

## In TP/SL Engine Specifically

The `tp_sl_engine.py` file:
- ✅ Does NOT assign confidence values
- ✅ Does NOT read or use confidence for exit decisions
- ✅ Uses regime, ATR, volatility, sentiment for TP/SL calculation
- ❌ Has no confidence gating on exits (only entries are gated)

Confidence is a signal-generation/entry metric, not a TP/SL metric.

---

## Summary

**0.7 confidence is a system-wide minimum threshold, not a hardcoded value.**

If you're seeing all signals at 0.7, it's because:
1. Agents are generating that value
2. Lower values are being filtered out
3. Or there's a default fallback in place

**Not a bug—this is by design.** Low-confidence signals are rejected.
