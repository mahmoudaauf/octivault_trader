🔥 CRITICAL FIX: Decision Generation Bug
==========================================

## Problem Summary
Signals were being generated by agents (6 signals in the last run) and execution requests were created (4 requests), but NO decisions were being built by MetaController. This resulted in:
- ❌ 0 trades executed
- ❌ decisions_count=0 
- ❌ 6 signals generated but discarded

## Root Cause
In `core/meta_controller.py` line 10948, the decision-building logic was checking **agent remaining budget** instead of the **signal's planned_quote**:

```python
# BROKEN CODE:
agent_budget = _wallet_budget_for(agent_name)  # Gets remaining budget (could be 0!)
if agent_budget >= significant_position_usdt:   # Fails if agent exhausted
    filtered_buy_symbols.append(sym)
```

**The Issue:**
- During allocation cycle, each agent gets allocated capital
- When `_build_decisions` runs, it checks if the agent still has remaining budget
- If the agent's budget was fully allocated, `agent_budget = 0`
- Check fails: `0 >= 25.0` is False
- Symbol is excluded from decisions
- Result: All signals filtered out → 0 decisions

## Solution
Changed the check to use the **signal's planned_quote** instead of remaining budget:

```python
# FIXED CODE:
signal_planned_quote = float(best_sig.get("_planned_quote") or best_sig.get("planned_quote") or 0.0)
if signal_planned_quote <= 0:
    signal_planned_quote = _wallet_budget_for(agent_name)  # Fallback only

if signal_planned_quote >= significant_position_usdt:  # Check signal size
    filtered_buy_symbols.append(sym)
```

**Why This Works:**
- Each signal carries its own `_planned_quote` from the allocation phase
- This represents what the agent INTENDED to allocate for this symbol
- Independent of whether agent has remaining budget
- Accurately reflects signal's minimum position size requirement
